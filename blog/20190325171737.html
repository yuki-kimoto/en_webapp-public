"Perl + Mojolicious" Web application development with Sakura VPS 7th User Creation

Finally, if you place a web application, execute it, and access it from a browser, I would like to reach the point where you can access the application.

The first thing you need to do is create a user. At the moment, you should see the following screen. 

#

If the line starts with # like this, it means that you are working as the root user.

It is absolutely necessary to avoid starting a web application as the root user. Because root privileges are very strong privileges. For example, you could delete all system files.

Therefore, it is very dangerous to run a web application with root privileges, and it is necessary to run it with lower privileges. 

Web applications run with lower privileges than root

So let's create a user to run the web application. The user name can be anything. I usually make it with a user named kimoto, but let's make it with your own name. Or maybe you could create a dedicated user for your web app.

This time, I would like to use the user name webapp.

Use the useradd command to create a user

Use the useradd command to create a user. 

useradd webapp

When executed, nothing is output on the screen, so I'm a little worried, but it's okay. On Unix / Linux, nothing is displayed when the process succeeds, and an error is displayed when the process fails, so remember to wake up. 

On Linux, if the command is successful, nothing is often output.

Use passwd to set a password for the user

Next, let's set a password for the user. Use passwd to set a password for the user. 

passwd username

Therefore, here, execute as follows. 

passwd webapp

You will be asked to enter your password. Enter your password and press Enter. 

Changing password for user webapp.
New password: New password:

The longer the password is, the safer it is, so use a password with at least 8 characters and a symbol. If it's too long, it's hard to remember and easy to forget, and if it's too short, it's dangerous. This is a feeling. Also make a note of this. It's especially easy to forget at first, and you'll forget it if you don't access it for a while. 

Make a note of the password as it is easy to forget.

After completing the first entry, you will be asked to enter the password again to confirm it, so enter it. Enter your password and press Enter. 

Retype new password:

It was successful when the following message appeared. 

passwd: all authentication tokens updated successfully.

User change is done with su

Now that we have created a user, let's change it to that user. Use the su command to change the user. Notice that there is a hyphen behind su. 

su --webapp

You can now change the user. Adding a hyphen after su means that it does not inherit the environment variables of the original user. For now, when switching users, keep in mind that environment variables are not inherited. 

The hyphen after su means that it does not inherit the environment variables of the original user

If you do not specify the user name as Tips, it means changing to the root user. 

<h3> Change to root user </h3>
su ---

Normally, when changing a user, the password of the user is required, but when changing from the root user to another user, the password is not required.

Return to original user

Use the exit command to return to the original user. 

exit

If you changed from root to a user called webapp, you can return to the root user.

Next time, I would like to actually run the web application.
